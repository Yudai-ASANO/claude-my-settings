# Performance Rules

## 基本原則

- 推測で最適化しない。計測してからボトルネックを特定する
- 可読性とパフォーマンスがトレードオフの場合、まず可読性を優先し、計測結果で必要と判明した箇所のみ最適化する

## パフォーマンス指針

- N+1 クエリを避ける。バッチ取得やプリロードを使用する
- 不必要な再計算を避ける。メモ化やキャッシュを検討する
- 大量データの処理にはページネーション/ストリーミングを使用する
- 非同期処理が可能な場合は並列実行を検討する
- ループ内でのオブジェクト生成・API 呼び出しを避ける

## データベース

- インデックスを適切に設定する
- 必要なカラムのみ SELECT する（`SELECT *` を避ける）
- トランザクションの範囲を最小限にする

## フロントエンド

- 初期ロード時に不要なデータを取得しない（遅延ロード）
- 大きなバンドルは分割する（コードスプリッティング）
- 画像やアセットは適切に最適化・圧縮する
