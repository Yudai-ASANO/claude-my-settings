# TDD（テスト駆動開発）セッションを開始してください。

## 対象: $ARGUMENTS

## 手順

Agent ツールで `tdd-guide` エージェントを起動し、以下のタスクを実行:

```markdown
## HANDOFF: orchestrator → tdd-guide

### タスク: $ARGUMENTS

### 指示: 以下の TDD サイクルで実装してください。

1. テスト戦略を策定（対象機能の分析、テストケース設計）
2. RED: 失敗するテストを書く
3. GREEN: テストを通す最小限の実装
4. REFACTOR: テストが通る状態でリファクタリング
5. 2-4 を繰り返す
6. テスト品質レビュー（網羅性、独立性、可読性）
7. カバレッジ確認（80%以上）
```

## TDD の原則

- テストを書く前に実装を書かない
- テストが失敗する理由を確認してから実装に進む
- GREEN フェーズでは最小限の実装のみ（過剰実装禁止）
- REFACTOR フェーズではテストが通ることを常に確認

## 完了条件

- 全テストが通る
- カバレッジ80%以上
- テスト品質レビューで問題なし
